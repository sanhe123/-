对nodeJS中path.resolve()解读

前言

  1. 如果你是先看了官网的解释仍不能理解
  2. 如果百度出了几篇解析文章和官网差不多一个意思
  3. 如果你仍然没放弃，恭喜你

定义

path.resolve() 方法将路径或路径片段的序列解析为绝对路径。

单刀直入

已知条件：我当前工作目录是 E:\self\nodejs\module

理解技巧：
1. 从右往左依次解析。（这句十分重要）
2. 代码里序号1-4的几种写法会返回当前工作目录。
3. 代码里序号5-8中，’net’ 等价于 ‘./net’ ：是相对路径，假如相对路径片段在最右侧，则需要向前拼接，
直到遇到以斜杠（/）开头的参数结束，假如直到解析到第一个参数仍然没有遇到以斜杠开头的，则直接拼接到你当前工作目录。
4. 代码里序号9-11说明的是参数从右至左，遇到第一个以斜杠开头的参数停止解析，无论前面是否还有参数。
5. ‘../net’ 解析之后就是net的上一级拼接net，
例如path.resolve(‘/img/icon’,’../net’) ：这里的net和icon是平级的，我们找到net的上一级是img，且img前有斜杠，说明img是根目录，拼接后结果就是（/img/net）
